Server Requirements

To run Task Closer your hosting server must support the following:

PHP version 5.4 or greater;
MySQL version 5.x or greater;
PHP PDO MYSQL extension;
PHP IMAP extension (for tickets autoimport);
PHP FSOCKOPEN extenstion (for service monitoring);
Cronjob (for automated tasks like email ticket and service monitoring).

Task Closer will run on almost all hosts, if monitoring doesn’t work contact your hosting provider for enabling php_fsockopen.

We strongly suggest that you enable SSL on your web server if you are going tu use the credentials manager feature.

Installation

Folow these steps to install Task Closer:

STEP 1. Upload Files

Connect to your web server with your favorite FTP client (ie. Filezilla) and upload the contents of the ‘upload’ folder.

STEP 2. Create the Database

Create a new MySQL database using your host’s control panel.

STEP 3. Run the Installation Script

Point a web browser to your domain plus “/install” (example http://www.my-domain.com/install). Enter your MySQL database configuration and enter the authentication details to create your first admin account. Click “Continue” to proceed to the next step.

If the database connection was successful the following screen will appear. Click “Install” to complete the installation.

STEP 4. Delete ‘install’ Directory

Using your FTP client, delete the ‘install’ folder from your web server.

To access Task Closer point your browser to your installation domain and login with the credentials provided during installation.

Notice: The installer needs to have permission to create the "config.php" file in the root directory of the installation.

Upgrading

Make a Full Backup - We cannot stress enough how important it is to *always* take a full backup before attempting any kind of upgrade. A full backup should be taken of both the Task Closer files and the database.


Follow these steps to upgrade Task Closer

STEP 1. Download

Download the new version and extract with your favorite file-archiver.

STEP 2. Backup

Backup all your files and your database.
You can use a FTP client to download all the files from your web server to your local computer.
For the database backup use the tools provided in the admin panel for your hosting account (usually PhpMyAdmin).

STEP 3. Upload

Upload the contents of the ‘upload’ folder to your web server with your favorite FTP client, overwriting all the files.
If you have made any custom modifications to Task Closer files, these will be overwritten.

STEP 4. Run the Upgrade Script

Point a web browser to your installation plus “/install/upgrade.php” (example http://www.my-domain.com/install/upgrade.php). This script will update the database to the required version.

STEP 4. Delete ‘install’ Directory

Using your FTP client delete the ‘install’ directory from your web server.

Cronjobs

To automatically import tickets from your IMAP mailbox you need to set the following cron job:
This cron job will check your IMAP account every 5 minutes for new tickets.

*/5 * * * * GET http://[replace with your url]/crons/tickets.php

For service monitoring to work you need to set the following cron job:
This cron job will run service monitoring tasks on all configured hosts every 5 minutes.

*/5 * * * * GET http://[replace with your url]/crons/monitoring.php

Configuration

After installation go to System > System Settings to configure Task Closer.

General Settings

Go to ‘General’ tab to set your company info, asset and license tag prefix, records to display per page in tables and a few other tweaks.

Email Settings

Go to ‘Email Settings’ tab to configure your mail server.
By default the app uses PHP mail to send mail.
You can configure a SMTP server if required.

SMS Settings

Service monitoring can send SMS alerts for enabled checks.
Go to SMS Settings tab to configure your SMS service provider.
Task Closer currently supports Clickatell and SMS Global.
You need to have an account with API access at one of this provider.

Notification Templates

In ‘Notification Templates’ tab you can customize the email and SMS templates Task Closer sends to users or admins.
Click on any notification to edit it’s content.
*SMS notifications are only used for monitoring alerts.

Tickets auto-import

Task Closer can automatically create tickets from an email message.
Go to ‘Tickets’ tab and add your IMAP mailbox authentication info.
You need to configure the imap.php cron job for this to work.
Each time the imap.php cron job runs it will covert any message found in ‘Inbox’ and purge that IMAP folder.

Cron Jobs

In the ‘Cron Jobs’ tab you will find the required information on how to set the required cron jobs for Task Closer to work properly.
Ticket auto-import requires the tickets.php cron job.
Service monitoring and alerting requires the monitoring.php cron job.

Key Aspects

ADMINS, CLIENTS & USERS

ADMINS can log in, have full access to the system and can create or edit other admins.
USERS can log în to the user panel, all user entities must be associated with a client, a user has access to all the information within the associated client.
CLIENTS cannot log in, can have multiple users that can log in to the user panel.

Example Let’s say that you are managing IT for Client Inc, within this client you add two users: Sarah from accounting and John from marketing, they both can access Task Closer to: submit tickets, view their company IT inventory and view work logs.

CUSTOMIZABLE DATA

In ‘System’ you can define the following data objects that will help you organize your data.
You can define objects like asset categories, license categories, status labels, manufacturers, asset models and suppliers. You will, later, use this objects to help you quickly create assets or licenses.

ASSETS

Overview of all added assets. Here you can add, edit and delete your assets.
Click on any asset to see or edit it’s details, work log through tasks, manage credentials and assign licenses.

LICENSES

Overview of all added licenses. Here you can add, edit and delete your licenses.
Click on any license to see it’s details and assign assets.

PROJECTS

Overview of all your projects. Here you can add, edit and delete projects.
Click on any project to see it’s details, add notes, assign issues, upload files, etc.
Use the issues board to visualiza and plan your work (Kanban Board).

TICKETS

Using tickets in Task Closer is very easy, you can create new tickets manually or import them automatically from an IMAP mailbox.
You can associate tickets to clients, users, admins and assets, this will allow you to track your work and report to your clients very easy.
Tickets can have one of the following states: Open, In Progress, Answered, Reopened or Closed.

ISSUES

Issues allow you to manage, track and plan your work.
You can associate issues with objects like projects, assets and/or clients, this will allow you to track your work and report yo your clients very easy.
Issues can have one of the following state: To Do, In Progress, Done.
Issues are can be categorized by type: Task, Bug, Maintenance, Improvement, New Feature. Use “Time Spent” option to track how much time you have worked on a issue.

MONITORING

Here you can add hosts for service monitoring.
Click on any host to add checks view current status and assign people to receive notifications.
You can assign multiple checks for one host. For example for host SERVER1 you can add a check to monitor the MySQL server and a check to monitor the web server.

REPORTS

Use this to generate reports for your clients.
Timesheet Generate a report with all the logged work within the given period for the selected client.
Inventory Generate a report with all assets and licenses for the selected client.
Timesheet with Inventory Generate a report with Timesheet + Inventory.

Using Task Closer

Translating

Task Closer uses an implementation of gettext to provide multilanguage support.

Gettext uses ".po" and ".mo" files to hold the translations.
".po" (Portable Object) is a text file that includes the original texts and the translations, this file can be used for editing.
".mo" (Machine Object) files are compiled and are easy for computers to read, this file is used by Task Closer to provide translations.

Follow these steps to translate Task Closer

  1. "template.po" file is provided in the "lang" folder
  2. Edit this file with a PO file editor
  3. Compile .mo file with your PO file editor, save your file in the following format languagecode.mo (example: en.po, fr.po, it.po etc.)
  4. Copy the .mo file in the "lang/overrides" folder of your Task Closer installation.
  5. Go to System > Settings > Localisation and add your new language, use the 2 letters language code for the code field.

To edit .po files you can use Poedit, a simple to use and free application.

Notice: As new features are added to Task Closer, new text strings might need translating, whith each new update, if new strings available, will be added to the template file. You can import them with Poedit in your language file.

Notice: Place all translation files in the "lang/overrides" folder. Placing them directly into the "lang" folder might get overwritten.

Notice: Language code must be the same as the filename (example: if you have the file fr.mo the code is fr).

Notice: If the system does not find the required string in your translation file it will use the default English for that string.

Notice: Users/Admins can set different languages, if preffered, which overrides the default language set in Settings > Localisation.
